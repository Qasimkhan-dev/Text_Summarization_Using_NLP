
# Model Logs

This folder contains the training and execution logs generated during different model experiments for the **Text Summarization Using NLP** project. These logs are used to track model behavior, training progress, errors, warnings, and evaluation-related details.

The logs help monitor how each summarization model performs during training, fine-tuning, zero-shot testing, and evaluation. They may include details such as training loss, validation loss, epoch progress, runtime messages, hardware usage, tokenization warnings, model loading status, and any errors that occurred during execution.

This folder is useful for analyzing experiments involving transformer-based summarization models such as T5, BART, PEGASUS, and other fine-tuned or zero-shot models. By reviewing these logs, it becomes easier to identify training issues, compare model stability, debug failed runs, and understand why a model performed better or worse.

The logs are also important for reproducibility because they provide a record of how each experiment was executed. They support comparison between different approaches, such as TF-IDF baseline, zero-shot XSum transfer, and fine-tuning on XSum.

Overall, this folder keeps the experiment history organized and helps explain the training and evaluation process of each summarization model.
